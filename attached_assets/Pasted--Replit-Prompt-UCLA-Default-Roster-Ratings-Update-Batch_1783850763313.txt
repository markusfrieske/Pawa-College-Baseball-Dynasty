## Existing Pitcher Storage Rules

The repo stores pitcher `velocity` as a 1-99 internal rating, then displays it as KMH through `velocityToKMH()`:

```ts
velocityToMPH(rating) = round(82 + ((rating - 1) * 20) / 98)
velocityToKMH(rating) = round(velocityToMPH(rating) * 1.60934)
```

So do not blindly put `151`, `146`, or `157` into `velocity`. Set the internal velocity rating that makes the UI display the screenshot KMH value, or the nearest possible value if the current conversion cannot display that exact number.

Recommended internal velocity targets with current conversion:

| Screenshot KMH | Internal `velocity` target | Display result |
|---:|---:|---:|
| 151 | 60 | 151 KMH |
| 146 | 45 | 146 KMH |
| 157 | 76 or 77 | 156 or 158 KMH |

For the 157 KMH pitcher, use the closest value and add a short code comment only if necessary. If exact 157 KMH display matters, adjust the display conversion separately and test every pitcher display that uses it.

## Pitch Mix Field Order

`pitchMix(1, [...])` uses this secondary pitch order:

```ts
pitchMix(1, [pitch2S, pitchSL, pitchCB, pitchCH, pitchCT, pitchSNK, pitchSPL, pitchFK, pitchSFF, pitchSHU])
```

These are the stored fields returned by `pitchMix`:

```text
pitchFB, pitch2S, pitchSL, pitchCB, pitchCH, pitchCT, pitchSNK, pitchSPL, pitchVSL, pitchFK, pitchSFF, pitchSHU
```

Important:

- `pitchFB` is binary. Fastball quality comes from `velocity`, `stuff`, and common abilities.
- `pitch2S`, `pitchFK`, and `pitchSFF` are binary in the helper.
- Most other pitch types use levels `0-7`.
- Match the screenshot pitch diagram as closely as possible.
