# Replit Handoff: Implement Generated Anime Art and Retro Icons

Use this document as the implementation direction for adding the already-generated art assets into the app.

Do not generate new images from this prompt. The images already exist. Your job is to place them in the codebase, wire them into the right screens, crop them well on desktop/mobile, preserve readability, and make the app feel more like a premium sports game companion.

## Product Goal

This app is a multiplayer college baseball dynasty companion for leagues that play games in eBaseball Power Pros. The artwork should make the app feel more game-like, immersive, and emotional while keeping the actual league-management UI fast and readable.

Most league members will use desktop, but every screen still needs to work well on phones.

## Design Principles

- Use art as atmosphere, not decoration that fights the UI.
- Desktop should feel rich and wide, like a sports game menu.
- Mobile should crop safely and keep the interface readable.
- Dense tables, stat forms, roster lists, and report-entry screens should sit on solid readable panels.
- Use image backgrounds mainly for hero areas, header bands, side panels, empty states, storyline thumbnails, modals, and major game moments.
- Keep the retro/pixel identity as flavor, but do not make the app harder to read.
- Do not replace normal UI icons like search, back, close, edit, upload, save, and settings. Use the generated retro icons only for game-specific concepts.

## Create the ArtworkBackground Component

Create:

```text
client/src/components/artwork-background.tsx
```

The component should support:

```ts
type ArtworkBackgroundProps = {
  desktopSrc: string;
  mobileSrc?: string;
  alt?: string;
  focalPoint?: string;
  overlayStrength?: "none" | "light" | "medium" | "heavy";
  className?: string;
  imageClassName?: string;
  children?: React.ReactNode;
  priority?: boolean;
};
```

Behavior:

- Render the image as an absolutely positioned background layer.
- Use `picture` or CSS media handling so mobile can use the mobile image when available.
- Use `object-fit: cover`.
- Use `object-position` from `focalPoint`.
- Add overlay classes based on `overlayStrength`.
- Keep children above the overlay.
- Add `aria-hidden="true"` if the image is decorative.
- If `alt` is provided and the image is meaningful, expose it appropriately without creating duplicate screen-reader noise.
- Do not block pointer events.
- Lazy-load non-hero images.
- Use `priority` or eager loading only for the landing hero.

Overlay guidance:

```text
none: no overlay
light: subtle dark gradient, good for decorative headers
medium: readable over most art
heavy: for text-heavy hero sections or mobile
```

## Performance Requirements

- Convert large PNGs to WebP or AVIF if the project supports it.
- Keep original PNGs only if needed.
- Use responsive images for hero backgrounds.
- Lazy-load non-critical art.
- Avoid importing every image into the landing bundle if that increases initial load too much.
- Consider route-level imports or a centralized registry that does not force all art into the first page bundle.
- Set fixed dimensions/aspect ratios for image panels to avoid layout shift.

Target behavior:

- Landing hero loads quickly.
- League pages do not feel sluggish.
- Mobile does not download unnecessary desktop-size images when avoidable.

## Accessibility Requirements

- If an image communicates page identity, use meaningful alt text.
- If an image is purely decorative, mark it decorative.
- Never place text over art without enough contrast.
- Support reduced motion.
- Keep all buttons and controls keyboard accessible.
- Icons must not be the only label for important actions.
