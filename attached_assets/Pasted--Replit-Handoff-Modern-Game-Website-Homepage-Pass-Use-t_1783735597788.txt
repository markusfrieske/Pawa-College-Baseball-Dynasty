# Replit Handoff: Modern Game Website Homepage Pass

Use this as the implementation brief for improving the homepage so it better represents the actual game/app.

The homepage currently has a strong visual direction, but the words and screenshots still make the product feel like a generic college baseball management sim. The real product is more specific and more compelling:

> A multiplayer college baseball dynasty league office for people who play games in eBaseball Power Pros, then recruit, report scores, track stats, manage storylines, and advance seasons together.

The homepage should feel like a modern game website: visual, confident, specific, and built around the actual weekly loop.

## Current Issues to Fix

### 1. The hero copy is too generic

Current copy:

```text
College Baseball Dynasty
The most immersive college baseball management sim. Recruit real talent, build your program, and compete against real coaches in a persistent league.
```

This is not wrong, but it undersells the unique product. It does not clearly communicate:

- Multiplayer league companion.
- Games are played externally in Power Pros.
- Weekly recruiting is the main game loop.
- Scores/box scores are reported after games.
- Commissioners can safely advance/rollback seasons.

Replace with a more specific game-site hero:

```text
Power Pros Dynasty League Office
Run an online college baseball dynasty around games played in eBaseball Power Pros. Recruit all week, play your series, report the box score, and advance the league together.
```

Alternative if we want to avoid leading with the Power Pros name:

```text
College Baseball Dynasty HQ
The online league office for multiplayer college baseball dynasties: recruit, schedule, report scores, track stats, and advance seasons with real coaches.
```

Recommended CTA buttons:

```text
Create League
Join League
Explore Demo
```

Secondary quick links should be:

```text
View Demo League
Browse Rosters
Create Recruiting Class
```

Do not make "Try as Guest" the strongest secondary CTA; "Explore Demo" feels more modern and product-led.

### 2. The homepage should show the weekly loop earlier

The current homepage talks about rosters, recruiting, and league competition, but it does not immediately show how someone actually uses the app.

Add a first section after the hero:

```text
Your Week in the Dynasty
1. Check your War Room
2. Recruit and scout prospects
3. Play your games in Power Pros
4. Upload or enter the box score
5. Ready up and advance safely
```

Make this section visual, with 5 horizontal steps on desktop and stacked cards on mobile.

### 3. Replace the conference ticker

Current ticker only lists conferences. That feels like a database site, not a living game.

Replace it with a live-style league ticker:

```text
FINAL: Oklahoma 6, North Carolina 4
REPORT PENDING: LSU vs Texas A&M
COMMITMENT: 4-star SS chooses Coastal Carolina
UPSET ALERT: #18 Oregon State falls in extras
ADVANCE READY: 11/14 coaches locked in
STORYLINE: Summer showcase stock rising
```

This makes the app feel alive even before users sign in.

### 4. Remove gem/bust spoiler language

The homepage currently says things like:

```text
generational gem
generational bust
Blue Chips, Gems, and Busts hidden in every class
```

That conflicts with the desired storyline/recruiting design. The site should sell uncertainty without spoiling labels.

Replace with:

```text
Every recruit starts uncertain. Scouting, storylines, showcases, injuries, and position changes reveal the player over time.
```

And:

```text
Some players rise, some fade, and some change paths completely before signing day.
```

### 5. Screenshots should show the actual companion-app loop

Current homepage screenshots:

```text
pipeline.jpg
player-card.jpg
commissioner.jpg
rankings.jpg
```

These are useful, but the homepage needs screenshots that tell the story of the app.

Recommended screenshot set:

1. **League War Room**
   - Shows next opponent, tasks, ready-up, league pulse.
   - Filename suggestion: `war-room.jpg`

2. **Recruiting Cockpit**
   - Shows board/battles/scouting/storyline uncertainty.
   - Existing `pipeline.jpg` can remain if no better screenshot exists.
   - Filename suggestion: `recruiting-cockpit.jpg`

3. **Game Reporting**
   - Shows screenshot upload/OCR/manual correction/box score workflow.
   - This is essential because it explains how Power Pros games connect to the app.
   - Filename suggestion: `game-reporting.jpg`

4. **Schedule / My Games**
   - Shows opponent preview and report status.
   - Filename suggestion: `schedule-my-games.jpg`

5. **Commissioner Safety**
   - Shows advance readiness, reports pending, save-state/rollback.
   - Existing `commissioner.jpg` can remain if updated.
   - Filename suggestion: `commissioner-safety.jpg`

6. **Stats / Record Book**
   - Shows league leaders, box score history, player stats, or records.
   - Filename suggestion: `stats-record-book.jpg`

Optional:

7. **Storylines**
   - Shows recruit/player event cards without revealing hidden gem/bust status.
   - Filename suggestion: `storylines.jpg`

### 6. The homepage currently repeats itself

There are multiple feature blocks that say similar things:

- Why College Baseball Dynasty
- Two-column image cards
- Real Results, Real Competition
- Why Coaches Choose Us
- Feature Deep-Dives
- Stats Banner

This can feel long and old-school SaaS. A modern game homepage should be tighter and more cinematic.

Recommended section order:

1. Hero
2. Live League Ticker
3. Your Week in the Dynasty
4. Feature Showcase with screenshots
5. Multiplayer League Tools
6. Recruiting and Storylines
7. Power Pros Game Reporting
8. Commissioner Safety and Save States
9. Stats, Records, and History
10. Final CTA
11. Footer

## Visual and Layout Direction

Make the homepage feel more like a modern game site:

- Use large cinematic screenshots with short captions.
- Use fewer, stronger sections.
- Use actual app screenshots, not generic stat claims.
- Use the generated hero/game art as atmosphere.
- Use motion subtly: ticker, screenshot carousel, hover transitions.
- Use stronger contrast and larger type for marketing copy.
- Reduce tiny pixel text in body sections.
- Keep pixel font for logo, badges, and flavor labels.
- Use readable modern sans-serif for paragraph copy and CTAs.
- Use desktop-first wide layouts, but keep mobile stacked and clean.

Avoid:

- Too many equal-weight feature cards.
- Long paragraphs.
- Dense stat claims too early.
- Gem/bust spoiler language.
- Newsletter form unless it actually works.
- Donate CTA in the primary marketing path.
